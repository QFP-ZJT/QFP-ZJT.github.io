设备参数

  1. ROM:25683
  2. RAM:256*8 据此PC8位,覆盖所有RAM区域

设计方案

硬件组成部分

CPU内部:

  1. 节拍发生器 不是流水线程序，所以没有必要严格按照节拍设计

  2. 寄存器组(PC(0) IR(1) MAR(2) MDR(3) 状态寄存器 累加器(4) 数据寄存器(5) 基址寄存器(6) R0(7))

    对外可见的只有累加(减/乘)器(R1) 数据寄存器(存储乘法超限的位数) 基址寄存器(为RAM使用) R0

  3. CU(指令译码与微程序实现)

  4. ALU模块

  5. 总线

CPU外部
  1. ROM(存储微程序)
  2. RAM(指令与数据)

设计部分

指令可以实现的功能
  • 取指
  • 从RAM中读
  • 向RAM中写
  • 两个数相加
  • 两个数相减
  • 两个数相乘
  • 寄存器间数据转移
  • 输出到指示灯
微程序指令实现部分 (由小节拍控制从ROM中读取指令控制执行,RAM默认读有效)
取值
  • T0:PC -> MAR
  • T1:M(MAR) -> MDR
  • T2:PC+1 -> PC
  • T3:MDR -> IR
从RAM中读 将IR或者其他寄存器中地址指定的数据传输到指定寄存器
  • T0:R0 -> MAR
  • T1:M(MAR) -> MDR
  • T2:MDR -> R1
向RAM中写 将R中的数据送入IR或者其他寄存器中地址指定的位置
  • T0:R0 -> MAR
  • T1:R -> MDR
  • T2:MDR -> M(MAR)
汇编与机器码对应关系
  • mov1 0001 对R1寄存器赋值
  • mov0 0010 对R0寄存器赋值

寻址方式

  • 立即数寻址 10
  • RAM寻址 11
  • 寄存器寻址 0~ 0或者1号寄存器
24位指令控制
  • [2:0]:后继微地址形成方式
    000备用 001顺序 010无条件跳转到[15：8] 011由IR高四位形成
  • [10:3]：没有想好
  • 需要添加的功能 cout什么时候接收 记录是否是相等的 中断申请位
  • [12:11]：写、读寄存器
  • [15:13]：选择寄存器
  • [17:16]：写、读开关
  • [20:18]：选择寄存器
  • [23:21]：ALU选择开关
