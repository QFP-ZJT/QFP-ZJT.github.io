零、管理员创建GitHub工程，并实现向服务器的同步工作(管理员已完成)

初始化

  • 在GitHub中创建一个repositorie
  • 获取项目的URL地址
  • 在服务器的网站根目录下创建git仓库git init
  • 连接远程的刚刚建好的GitHub仓库git remote add origin 项目的URL

    常用的操作命令

  • git remote -v查看连接的远程仓库
  • git push origin master将本地的代码提交到GitHub仓库
  • git add .将所有文件放进新的本地git仓库中
  • git commit -m "备注" 将添加的文件提交到仓库
    “常用 Git 命令清单”

一、组队

  • 在github中创建一个组织
  • 邀请队员加入，并给队员最高的权限相信大家啊

二、从GitHub上下载项目

  • 在PhpStorm中找到/VCS/checkout from version control/Github选项
  • 在弹出的文本框中的Git Repository URL中填入https://github.com/QLsoftware/qlsoftware/第二行选择存放于本地的哪个目录下，第三行为项目的名称，我们统一称为qlsoftware就可以了
  • 点击clone即可将项目下载到本地
  • 你可能遇到的问题:在下载过程中你可能会发现缺少一部分文件，这个时候，你只需要找到项目的根目录下，找到.gitignore文件，并清空里面的内容。问题应该就会解决

项目已经初始化，请以Github上的作为标准

三、本地与远程数据库之间的数据同步

简述

在/VCS/git菜单下可以看见很多功能，这些功能之间的关系如图.引用于“Git远程操作详解”

简单的总结

  • fetch:用于更新自己的本地代码。将其他队员的提交，更新到自己的本地
  • push:将本地的代码提交到远程代码库，有两种选择，分支或者直接合并到master中(相当于直接更改)在push之前需要将代码commit，即将代码先放入本地仓库中
  • clone:从远程代码库下载整个文件到本地

补救方法

  • 最好还是每个人独有自己的文件，不要共同编写一份文件，还没有搞明白同一个文件刷新的时候是怎么做的。
  • 万一自己写的代码乱掉了，不要慌张，在项目上右键，找到Local History Show History，检查自己的操作记录，恢复自己的代码。
  • 还是多写点注释比较好，特别是在上传自己的代码的时候。多写注释，别惹队友手中的菜刀。
